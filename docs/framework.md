# 四能递进式课程重构模型

## 1. 模型定位

“四能递进式课程重构模型”面向高职和应用型本科工程技术类课程，旨在解决学生“会听、会背、会模仿，但不会从真实任务建立专业模型并完成工程判断”的问题。

模型强调：

```text
真实任务输入 → 结构翻译 → 模型建立 → 过程求解 → 工程判断 → 能力输出
```

## 2. 四类能力定义

| 能力 | 定义 | 典型证据 |
|---|---|---|
| 结构翻译能力 | 把真实设备、工艺对象、任务场景或故障现象翻译成专业结构、变量参数、功能关系、图表、符号或模型 | 结构标注图、变量表、接口关系图 |
| 运动建模能力 | 根据结构、任务和约束，建立可用于分析、计算、仿真、编程或控制的专业模型 | 机构模型、控制模型、工艺模型、仿真模型 |
| 过程求解能力 | 按“已知条件—目标结果—方法选择—分步实施—结果验证”的流程完成任务 | 计算书、程序、调试记录、仿真报告 |
| 工程判断能力 | 判断方案、程序、模型或结果是否满足工程现场要求 | 可行性判断表、风险分析表、优化建议 |

## 3. 四能之间的递进关系

### 3.1 结构翻译是入口

学生首先要看懂对象，把“中文任务”“设备结构”“工艺现象”翻译为专业变量、图表和模型对象。

### 3.2 运动建模是核心

学生需要把翻译后的结构、变量和约束建立为可分析、可计算、可仿真的模型。

### 3.3 过程求解是路径

学生需要遵循稳定流程，而不是凭经验碰运气：

```text
已知条件 → 目标结果 → 方法选择 → 分步实施 → 结果验证
```

### 3.4 工程判断是落点

工程教育不能只看“算出来”，还要判断“能不能用”：安全、精度、效率、限位、干涉、稳定性、成本、节拍都要纳入判断。

## 4. 课程改造通用流程

1. 提取课程知识点和技能点；
2. 识别岗位任务和典型工程场景；
3. 将课程内容映射到四类能力；
4. 设计递进式项目；
5. 为每个项目配置四能任务链；
6. 设计学习成果证据；
7. 形成评价量规；
8. 输出课程标准、任务书、PPT大纲、课堂任务单和评价表。

## 5. 推荐课堂任务单

| 步骤 | 学生任务 | 输出结果 |
|---|---|---|
| 1. 看对象 | 观察设备、结构、工艺或现象 | 对象描述 |
| 2. 找组成 | 识别部件、接口、流程、信号 | 结构组成表 |
| 3. 定变量 | 明确输入量、输出量、参数和约束 | 变量参数表 |
| 4. 建模型 | 建立机构、控制、工艺、数学或仿真模型 | 模型图/模型文件 |
| 5. 走流程 | 按步骤完成计算、编程、调试或仿真 | 过程记录 |
| 6. 做判断 | 判断安全、精度、效率、限位和可行性 | 工程判断表 |

## 6. 课程建设口号

> 从“会听会算”转向“会看、会建、会解、会判”。
